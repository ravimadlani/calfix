health_score_factors NEW - Configuration
Master configuration table for all health score factors. Allows enabling/disabling factors and adjusting their point values without code changes.
Column Type Constraints Description
id UUID PRIMARY KEY Unique identifier
factor_key TEXT NOT NULL UNIQUE Unique key (e.g., 'back_to_back', 'double_booking')
factor_name TEXT NOT NULL Display name for UI
description TEXT Explanation of what this factor measures
category TEXT Category: scheduling, meeting_hygiene, work_life_balance, travel
impact_type TEXT NOT NULL 'per_occurrence' or 'threshold'
base_impact INTEGER NOT NULL Points to add/subtract (negative for penalties)
threshold_value DECIMAL For threshold types (e.g., 6 for hours)
threshold_operator TEXT Comparison operator: 'gt', 'lt', 'gte', 'lte', 'eq'
is_active BOOLEAN DEFAULT true Whether to include in calculations
is_positive BOOLEAN DEFAULT false true for bonuses, false for penalties
applies_cumulative BOOLEAN DEFAULT true Whether multiple occurrences stack
allow_user_override BOOLEAN DEFAULT false Can users customize this factor?
min_impact INTEGER Minimum value users can set
max_impact INTEGER Maximum value users can set
user_health_factor_overrides NEW - User Customization
Allows users to customize health factor weights for their personal preferences (if enabled by admin).
Column Type Constraints Description
id UUID PRIMARY KEY Unique identifier
user_id TEXT NOT NULL FK ‚Üí users.id User who customized the factor
factor_id UUID NOT NULL FK ‚Üí health_score_factors.id Factor being customized
custom_impact INTEGER User's custom point value
is_disabled BOOLEAN DEFAULT false User disabled this factor
created_at TIMESTAMPTZ DEFAULT NOW() When customized
updated_at TIMESTAMPTZ DEFAULT NOW() Last update time
